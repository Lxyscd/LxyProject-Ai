### ARCHITECTURE_DETAILS - "怎么做的"

**最重要的 3 个代码模式**：

```java
// 1. 多模型故障转移
try { return callQwen(...); }
catch { return callZhipu(...); }

// 2. 异步处理
@Async
public void processAnalysis(...) { /* 后台处理 */ }

// 3. 流量限制
if (!rateLimiter.isAllowed(user)) {
    return 429 Too Many Requests;
}
```

**最重要的 3 个架构决策**：
1. 为什么用异步？→ 避免阻塞，支持高并发
2. 为什么本地 NIMA？→ 降低成本，增强控制
3. 为什么多模型？→ 提高可用性，优化成本
