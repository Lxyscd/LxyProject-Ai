# 🎓 本科毕设项目：LenseMaster AI 摄影美学评论系统

## 📋 项目概述

**项目名称**: LenseMaster AI - 智能摄影美学评论与分析平台

**项目类型**: 全栈 AI 应用

**核心功能**: 利用深度学习与多模态大模型，为摄影师提供专业的美学评论、构图分析和拍摄建议

---

## ✨ 项目核心亮点

### 1. **多模态 AI 融合技术栈** 🤖
- **NIMA 审美评分模型**：基于 MobileNet 架构的深度学习模型，对照片进行客观的美学评分（0-10分）
- **多大模型融合**：
  - **Zhipu GLM-4V**（智谱）：国产多模态大模型，视觉理解能力强
  - **Qwen VL-Max**（阿里）：通义千问视觉模型，作为备用方案
  - **Google Gemini**（可选）：多语言支持
  - 实现 **故障转移机制**：Qwen 失败自动切换到 Zhipu，确保服务高可用
- **无缝融合**：将 NIMA 的定量评分与 AI 的定性分析相结合，生成多维度评论

### 2. **业界级架构设计** 🏗️
- **微服务架构**：Java Spring Boot + Python Flask + React 前端的三层分离
- **异步任务处理**：
  - 使用 `@EnableAsync` 实现非阻塞分析
  - 支持长任务与实时进度查询
  - 任务队列管理，避免服务雪崩
- **智能缓存与相似度检索**：
  - 基于向量相似度搜索历史分析
  - 快速定位相似作品（5 条相关照片推荐）
- **速率限制**：单个用户限流（1分钟内的请求限制），防止 API 滥用

### 3. **专业级摄影学分析维度** 📸
AI 系统生成的分析包括 9 个专业维度：

| 维度 | 说明 | 应用场景 |
|------|------|--------|
| **构图 (Composition)** | 分析线条、平衡、视觉重心及构图法则 | 协助学习黄金分割、三分法等 |
| **光影 (Lighting)** | 评价光线质感、方向、对比度及氛围 | 指导室内/室外光源选择 |
| **色彩 (Color)** | 探讨色调一致性、饱和度及情感传达 | 色彩搭配优化建议 |
| **综合评价 (Overall)** | 富有诗意的总结 | 激励用户创作 |
| **进阶建议 (Suggestions)** | 3 条具体的拍摄或后期建议 | 实战操作指导 |
| **二次构图 (Crop)** | 给出最优裁剪方案（坐标+理由） | 后期处理参考 |
| **主体分析 (Subject)** | 识别并描述照片主体，支持**点击交互** | 突出用户关注的区域 |
| **参数估计 (EXIF)** | 根据视觉特征推断拍摄参数 | 复现拍摄条件 |
| **多维评分 (Scores)** | 5 个维度的 0-100 评分 | 数据可视化与对比分析 |

### 4. **交互创新** ✨
- **点击交互分析**：用户可点击照片任意位置，AI 优先分析该区域的主体
- **数据可视化**：
  - 趋势图：最近 7 次分析分数变化
  - 雷达图：5 维度评分展示
  - 统计仪表板：总分析数、平均分等
- **社交分享**：拍照 → 分析 → 分享，一键复制评论结果

### 5. **完整的数据管理系统** 💾
- **分析历史**：永久保存用户的每一次分析结果
- **用户系统**：
  - 基于 JWT 的身份认证
  - Spring Security 集成
  - 用户隔离与数据隐私保护
- **作品库（Gallery）**：用户可分享分析作品，建立个人摄影集
- **评论系统**：用户可对分享作品进行评论与点赞

### 6. **工业级部署与容器化** 🐳
- **Docker Compose 一键部署**：完整的微服务编排
  - MySQL 数据库容器
  - Spring Boot 后端容器
  - Python NIMA 服务容器
  - React 前端容器
- **环境隔离**：`.env` 文件管理 API Key，生产/开发环境配置分离
- **跨域支持**：CORS 配置支持前后端分离部署

### 7. **健壮的错误处理与日志系统** 🛡️
- **全局异常处理**：`GlobalExceptionHandler` 统一捕获与响应
- **系统日志服务**：
  - 持久化存储操作日志
  - 支持 INFO/WARN/ERROR 等级别
  - 完整的错误追踪栈
- **API 降级方案**：
  - 当 NIMA 服务不可用时，返回随机合理分值（6.5-8.0）
  - 当 AI 分析失败时，返回友好的错误提示与备用建议

### 数据流程：从上传到结果呈现

```
用户上传图像
    ↓
─────────────────────────────────────────────────────────
前端阶段：
  1. 图像压缩 & base64 编码
  2. 提取 EXIF 元数据（快门速度、光圈、ISO等）
  3. 计算 CV 特征（色彩分布、边缘能量、纵横比）
  4. 生成向量表示 (用于相似度搜索)
─────────────────────────────────────────────────────────
    ↓
后端接收：创建分析任务（PENDING 状态）
    ↓
─────────────────────────────────────────────────────────
异步处理链（AsyncTask）：

  [Step 1] NIMA 评分
    └─→ 调用 Python 服务
    └─→ 返回美学评分 (0-10)

  [Step 2] CV 特征提取
    └─→ 计算色彩、构图复杂度等

  [Step 3] AI 多维分析
    └─→ 构建精细 prompt（包含 NIMA 分数+CV特征）
    └─→ 尝试调用 Qwen VL
    └─→ Qwen 失败 → 回退到 Zhipu GLM-4V
    └─→ 返回 9 维度 JSON 分析结果

  [Step 4] 向量化存储
    └─→ 计算图像特征向量
    └─→ 用于后续相似度搜索

  [Step 5] 相似作品推荐
    └─→ 从历史库检索 Top-5 相似作品
    └─→ 返回给前端
─────────────────────────────────────────────────────────
    ↓
更新分析记录（COMPLETED 状态）
    ↓
前端实时轮询查询状态
    ↓
获取完整结果并展示
```

## 🔑 核心算法与创新点

### 1. **多模型融合评分算法**
```
最终评分 = NIMA分数 + AI定性分析
  ├─ 定量部分：NIMA (0-10)
  │  └─ 基于深度学习，客观无偏
  └─ 定性部分：9 维度 AI 分析
     ├─ 构图、光影、色彩等专业维度
     ├─ 进阶建议与参数估计
     └─ 支持点击交互的主体识别
```

### 2. **故障转移与服务高可用**
```python
# 伪代码
analyze_image():
    try:
        return call_qwen(image, prompt)  # 优先用通义千问
    except QwenError:
        logger.warn("Qwen failed, fallback to Zhipu")
        return call_zhipu(image, prompt)  # 备用方案
    except ZhipuError:
        logger.error("All AI models failed")
        return build_fallback_response()  # 降级方案
```

### 3. **异步任务队列**
```
用户提交 → 立即返回任务ID → 前端轮询查询
优点：
  ✓ 不阻塞用户请求
  ✓ 长任务处理优雅
  ✓ 支持并发分析
```

### 4. **向量相似度搜索**
```
相似度计算：
  1. 提取图像特征向量 (从 CV 特征 → embedding)
  2. 计算向量余弦距离
  3. 返回 Top-5 最相似作品
应用：
  ✓ "我的摄影风格相近作品有哪些？"
  ✓ 发现学习标杆
```

### 5. **提示词工程 (Prompt Engineering)**
- **精细的上下文注入**：NIMA 分数 + CV 特征 → 更准确的 AI 分析
- **点击位置感知**：用户点击 → 系统优先分析该区域
- **JSON 格式约束**：强制 AI 输出结构化数据
- **中文 + 摄影术语**：使用专业摄影词汇提升分析质量
